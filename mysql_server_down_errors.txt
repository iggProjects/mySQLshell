  IMPORTANT: do NOT delete ibdata1 file. You could destroy all your databases.

  Instead, first try using the MySQL backup folder which is included with XAMPP. So do next steps:

  1.- EN PHMYADMIN/CONFIG.INC.PHP, CAMBIAR CLAVE POR '' PARA USUARIO ROOT EN LOCALHOST, y poner "AllowNoPassword" true
      ( En /localhost/mysqlshell/config.servers hacer lo mismo )
  2.- Rename folder mysql/data to mysql/data_old
  3.- Make a copy of mysql/backup folder and name it as mysql/data
  4.- Copy all your database folders from mysql/data_old into mysql/data (except mysql, performance_schema, and phpmyadmin folders)
  5.- Copy mysql/data_old/ibdata1 file into mysql/data folder
  6.- Start MySQL from XAMPP control panel

  LUEGO DE REALIZAR LOS PASOS ANTERIORES, HACER:  
  1.- INGRESAR EN "phpmyadmin";
  2.- REPARAR EN MYSQL LA TABLE "DB";
  3.- ASIGNAR CLAVE A ROOT VIA PHPMYADMIN COLOCAR ESA CLAVE DE ROOT EN LOCALHOST Y 127.0.0.1 DEL CONFIG.SERVERS.PHP, 
      y poner "AllowNoPassword" false
  4.- COLOCAR ESA CLAVE DE ROOT LOCALHOST EN EL FILE PHMYADMIN/CONFIG.INC.PHP, y poner "AllowNoPassword" true

# Based on this answer: https://stackoverflow.com/a/61859561/1956278

# Backup old data
Rename-Item -Path "./data" -NewName "./data_old"

# Create new data directory
Copy-Item -Path "./backup" -Destination "./data" -Recurse
Remove-Item "./data/test" -Recurse
$dbPaths = Get-ChildItem -Path "./data_old" -Exclude ('mysql', 'performance_schema', 'phpmyadmin') -Recurse -Directory
Copy-Item -Path $dbPaths.FullName -Destination "./data" -Recurse
Copy-Item -Path "./data_old/ibdata1" -Destination "./data/ibdata1"

# Notify user
Write-Host "Finished repairing MySQL data"
Write-Host "Previous data is located at ./data_old"
